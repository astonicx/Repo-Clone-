# GITHUB COPILOT PROMPT — "FULL GAME CLONE SYSTEM"

You are assisting in creating a full GitHub-based clone of an existing game.
Your job is to generate code, documentation, and step-by-step systems that recreate the game's mechanics with new original code.
Follow every instruction below exactly.

## 1. PROJECT GOAL

Rebuild the entire game inside this repository with:

- The **same mechanics**
- The **same systems**
- The **same gameplay loop**
- Fully rewritten **original code**
- Clean, modular, documented architecture
- A step-by-step development workflow

This repo must become a **complete, detailed, intricate clone** of the game's behavior.

## 2. WHAT YOU MUST GENERATE

Copilot must help produce:

### A. Game Systems Documentation

Create/maintain a file named `GAME_SYSTEMS.md` containing:

- Core gameplay loop
- Player mechanics
- Movement rules
- Collision rules
- Scoring system
- Level progression
- UI/HUD behavior
- Game states (start, pause, game over)
- Any additional mechanics the original game has

Each mechanic must include:

- Description
- Technical breakdown
- Edge cases
- Pseudocode
- File(s) responsible for implementation

### B. Step-by-Step Build Plan

Create/maintain a file named `BUILD_STEPS.md` with:

1. Repo setup
2. Environment setup
3. Branching strategy
4. Feature-by-feature implementation order
5. Testing steps
6. Commit guidelines
7. Pull request workflow

Each step must be actionable and numbered.

### C. Code Implementation

For each mechanic:

- Create a feature branch
- Generate clean, modular code
- Add comments explaining logic
- Follow the structure defined in `GAME_SYSTEMS.md`
- Ensure the mechanic behaves exactly like the original game

### D. Developer Log

Maintain a file named `DEV_LOG.md` where each entry includes:

- Date
- Feature branch
- What was implemented
- What changed
- What still needs refinement

## 3. REPO STRUCTURE TO FOLLOW

```
/src
  /components
  /systems
  /logic
  /ui
  /utils
  /hooks
/docs
  GAME_SYSTEMS.md
  BUILD_STEPS.md
  DEV_LOG.md
/tests
README.md
package.json
```

## 4. BRANCHING RULES

Copilot must always suggest branches like:

- `feature/movement`
- `feature/collision`
- `feature/scoring`
- `feature/ui`
- `feature/game-state`

Each branch must include:

- Small, logical commits
- Clear, descriptive commit messages
- A pull request summary explaining what changed

## 5. IMPLEMENTATION STYLE

Copilot must:

- Use clean, readable code
- Avoid unnecessary complexity
- Add comments for every non-obvious line
- Follow React/TypeScript best practices
- Keep logic modular and testable
- Keep UI separate from game logic
- Keep game state isolated and predictable
- Suggest improvements when needed

## 6. WHAT COPILOT SHOULD NEVER DO

- Never copy original game code verbatim
- Never copy original copyrighted assets
- Never use copyrighted text or dialogue
- Never skip documentation
- Never merge large unreviewed changes
- Always recreate mechanics with original implementations

## 7. WHAT TO DO FIRST

When starting a new mechanic or feature:

1. Reference `GAME_SYSTEMS.md` for specifications
2. Create a feature branch following naming conventions
3. Generate or update relevant documentation first
4. Implement code in logical, testable chunks
5. Write meaningful commit messages
6. Create a pull request with clear description

## 8. TONE & OUTPUT STYLE

Copilot must:

- Be extremely detailed
- Be structured and organized
- Be technical and precise
- Provide explanations AND code
- Show edge cases and error handling
- Include comments in code
- Link to relevant documentation files

## 9. WHEN GENERATING FILES

Always ask or clarify:

- What game mechanic or system is being implemented?
- What should the technical specifications be?
- What edge cases need to be handled?
- How does this integrate with existing systems?

## 10. QUALITY CHECKLIST

Before suggesting code is "done", ensure:

- [ ] Code runs without errors
- [ ] Mechanic matches original game behavior
- [ ] Code is readable and commented
- [ ] Tests are written or suggested
- [ ] Documentation is updated
- [ ] No copyrighted material is used
- [ ] Commit message is clear
